# Queues — divergence (docs/ ↔ REFACTOR.MD ↔ src/types)

> Parked working note. Goal: decide the canonical API for queues, then reconcile all three sources. No source is authoritative yet.

## Three-source status
- **docs/queues.md**: Conceptual doc; client-side `registerHandler` + `deadLetters` query/handle API; enqueue options `priority`/`ttlSeconds`/`ScheduledDeliveryOptions`; `DEAD_LETTER` sentinel, `MANUAL`/`UnrecoverableError` explicitly excluded for queues.
- **REFACTOR.MD Part 10 (Queues)**: Older, richer spec; richer `defineQueue` config (`defaultTTL`, `retention`), full TTL/scheduling/priority semantics, Postgres schema, claim SQL, retry-TTL-reset behavior; only `max_attempts`/`ttl_expired` dead-letter reasons.
- **src/types (impl)**: Partial. Definition-level `registerHandler` is a no-op stub (`messaging.ts`); the real client surface (`QueueNamespaceExternal.registerHandler` + `DeadLetterNamespaceExternal`) is typed in `engine.ts` and stubbed in `client.ts` (throws "Not implemented"). `defineQueue` accepts only `{ name, message, ttlSeconds? }`. `DEAD_LETTER` is real; `DeadLetterReason` adds a third value `"unrecoverable"`.
