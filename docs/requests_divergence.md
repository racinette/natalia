# Requests — divergence (docs/ ↔ REFACTOR.MD ↔ src/types)

> Parked working note. Goal: decide the canonical API for requests, then reconcile all three sources. No source is authoritative yet.

## Three-source status
- **docs/requests.md** (+ **resolving-requests-asynchronously.md**): Conceptual doc; forward handlers registered on the client via `client.registerRequestHandler(def, handler, opts)` with `retryPolicy`/`onExhausted`; `MANUAL` returned by handler or `onExhausted`; compensation via separate `registerRequestCompensationHandler`; call site `(payload)` or `(payload, { priority, timeout })` where `timeout` is required when options are passed.
- **REFACTOR.MD Part 11 (Requests)**: Older, richer spec; `defineRequest` uses **`request:`** (not `payload:`) + `retention`; client `registerRequestHandler` with `totalTimeoutSeconds` + `onExhausted` (default `MANUAL`); full `RequestCompensationInfo`/`registerRequestCompensationHandler` signature, compensation query surface, Postgres schema, `resolve`/`cancel` semantics.
- **src/types (impl)**: Partial. `defineRequest` returns a definition with a **no-op stub `registerHandler` on the definition** (`requests.ts:52`, `workflow.ts`). **There is no forward `registerRequestHandler` anywhere** — not on the client, not exported. Only `registerRequestCompensationHandler` is implemented/exported (`workflow.ts:533`, `index.ts:57`). `MANUAL` is defined as the **compensation** sentinel (`workflow.ts:75-78`). Manual-resolution client surface (`RequestNamespaceExternal`, `RequestHandleExternal.resolve/cancel`) is typed (`engine.ts`) and stubbed (throws "Not implemented", `client.ts:94-131`).
